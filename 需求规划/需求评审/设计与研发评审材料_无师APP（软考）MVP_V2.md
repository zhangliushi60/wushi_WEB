# 设计与研发评审材料：软考AI私教MVP V2.1

> 评审日期：2026年7月4日
> 评审对象：PRD V2.1（高项方向 + 千人千面AI备考）

## 一、AI论文批改技术评估

- 需30篇论文双盲测试验证评分偏差
- 查重需自建模板库+公开范文库
- 论文框架生成教研认可率目标 ≥85%
- 逐段精批：整篇处理 vs 分段处理各有优劣
- 异步队列方案：Redis/RabbitMQ + 推送通知

## 二、大模型API选型

| 模型 | 综合知识解析 | 案例分析 | 论文批改 | 成本 |
|------|:-----------:|:--------:|:--------:|:----:|
| GPT-4o | 优秀 | 优秀 | 优秀 | 高 |
| Claude 3.5 | 优秀 | 优秀 | 优秀 | 中高 |
| DeepSeek V3 | 良好 | 良好 | 待验证 | 低 |

## 三、技术架构

客户端(React Native) → API Gateway → 业务服务层(用户/题库/刷题/诊断/推荐) → AI服务层(解析/案例/论文) → 数据层(MySQL/Redis/OSS)

## 四、核心数据表

questions, question_tags, ai_explanations, user_preferences, study_plans, study_tasks, user_answers, error_book, essay_submissions, essay_reviews, case_submissions

## 五、页面清单（24页）

公共(3页)：启动页、注册登录、备考引导
首页(1页)：首页（AI计划卡片+三科入口+统计）
综合知识(4页)：章节列表、刷题页、解析页、AI追问
案例分析(3页)：案例列表、答题页、评估页
论文(5页)：题目列表、框架页、撰写页、批改结果、素材推荐
通用(4页)：错题本、雷达图、每日一练、模拟考试(1.0)
用户(2页)：会员购买、个人中心

## 六、技术风险

- AI论文批改准确率不达标 → 2周PoC验证
- 8周排期过紧 → 准备Plan B砍案例
- 教研团队未到位 → 立即启动招聘
- API成本超支 → 多模型分流+缓存+配额
